Introduction

The z11 package provides an R-interface to the geospatial raster data of the German Census 2011. z11 can be used to list all available attributes from the Census (1 km² attributes and 1-hectare attribute) and import them either as simple features spatial points object or as raster object with the resolution of the corresponding attribute. I felt that such a package may be of interest for other userRs since the Census data ‘only’ exists as CSV data dump on the internet. However, to use them in a Geographic Information System (GIS), the data must be prepared, which is time-consuming and computationally demanding. As such, the package aims to fill the gap of making the data easily accessible by providing a straightforward method.

Generally, the German Census 2011 data are available under a Data licence Germany – attribution – version 2.0 and can be manipulated and openly shared. Yet, as part of this package, use them at your own risk and do not take the results of the functions for granted.

Installing and Loading the Package

remotes::install_github("StefanJuenger/z11")

After installing, the package can be loaded using R’s standard method:

library(z11)

Downloading the Census data

Since the Census data is only available in very large .csv files that are difficult to handle, this package relies on pre-prepared data. Here, you have two options: either you save the Census data in .rds format, or store it in a SQL database. When joining a large number of Census variables to your data, using a SQL database can be faster, and less computationally expensive. Though for the most part, using the .rds data might be more convenient.

.rds data

You can directly download the pre-prepared data from this github repository. This data was downloaded from the Census website, unzipped and pre-prepared using the z11_prepare_data() function, although there is no need to do this yourself.

z11_prepare_data("/home/yourname/census_csv", "/home/yourname/z11data")

SQL database

If you wish to store the Census data in a SQL database, you first have to download and unzip the data. If you’re using Linux, you can use the z11_download_data() function for this, otherwise download the .csv data from the Census website and unzip it manually. Next, use the z11_prepare_data() function to read in the .csv data, transform it, and store it in your database.

z11_download_data("/home/yourname/census_csv")

library(DBI)
con <- dbConnect(RSQLite::SQLite(), "/home/yourname/z11data.sqlite3")
z11_prepare_data("/home/yourname/census_csv", con)

Working with 1 km² German Census 2011 Data

1 km² data were the first data published in 2015. While they comprise fewer attributes than the 1 hectare ones (see below), the data are in a way more easy to use format (see also the z11 package’s vignette about the initial data preparation [still in the works]). You can browse all available attributes by using the following function:

z11::z11_list_1km_attributes()
#>  [1] "ab65_A_cat"          "ab65_A"              "Alter_D_cat"        
#>  [4] "Alter_D"             "Auslaender_A_cat"    "Auslaender_A"       
#>  [7] "Einwohner_cat"       "Einwohner"           "Frauen_A_cat"       
#> [10] "Frauen_A"            "HHGroesse_D_cat"     "HHGroesse_D"        
#> [13] "Leerstandsquote_cat" "Leerstandsquote"     "unter18_A_cat"      
#> [16] "unter18_A"           "Wohnfl_Bew_D_cat"    "Wohnfl_Bew_D"       
#> [19] "Wohnfl_Whg_D_cat"    "Wohnfl_Whg_D"

Thus, in sum we can use 20 different census attributes of the size of 1 km². A full list of the variables can also be found in the z11 variables vignette. Please note that some of the attributes are also duplicated, denoted by the _cat-suffix in the attribute name. These attributes may be less precise, but they comprise fewer missing values than the continuous attributes. In any case, for more details about the actual attributes, refer to the official documentation of the German Census 2011 at https://www.zensus2011.de.

Now, if we want to load one of the attributes as a raster layer, we can use the z11::z11_get_1km_attribute() function. For example, for importing information on immigrant rates on a 1 km² grid level, we could use the following command:

immigrants_1km <- z11_get_1km_attribute("Auslaender_A", data_source = "/home/yourname/z11data")

If you’re working with a SQL database, pass your DBI Connection to the data_source argument:

immigrants_1km <- z11_get_1km_attribute("Auslaender_A", data_source = con)
#> Fetch attribute from database...
#> Extract coordinates from inspire ID...
#> Transform to raster...

Voilà, we got this information as a standard raster layer (class("Raster")) as specified in the raster package:

Therefore, we can also use the raster package’s standard plotting procedures:

raster::plot(immigrants_1km)

Suppose we do not want to work with raster data. Instead, we aim to use the raster grid cell’s centroid coordinates. In that case, we can also load the data as a simple features data frame with point geometries as specified in the sf package by simply using the option as_raster:

immigrants_1km_sf <- z11_get_1km_attribute("Auslaender_A", data_source = con, as_raster = FALSE)
#> Fetch attribute from database...
#> Extract coordinates from inspire ID...

Working with 1 hectare German Census 2011 Data

In 2018, destatis also published Census data on a 1-hectare grid level. This 100m \(\times\) 100m data provides a more fine-grained level of information for Germany’s population. Moreover, the data comprise way more attributes than the 1 km² one, including details on buildings and even heater types. Again, we can list all available attributes using a specialized function:

head(z11_list_100m_attributes())
#> [1] "DEM_ALTER_10JG_1" "DEM_ALTER_10JG_2" "DEM_ALTER_10JG_3" "DEM_ALTER_10JG_4"
#> [5] "DEM_ALTER_10JG_5" "DEM_ALTER_10JG_6"

The number of attributes is quite high, in sum 229. The naming convention is different as the names depict a concatenated string between a three character prefix representing the source .csv dataset, each attribute name and its category. A full list of the available attributes with short descriptions can be found in the z11 variables vignette.

In contrast to the 1 km² data, the attribute values are the number of persons within each 1-hectare grid cell sharing the attribute’s characteristic. Again, please refer to the official German Census 2011 documentation since it details how to handle the data and interpret its values. The data here are just vanilla Census data. However, they were converted from a long data format to a wide one for rasterizing.

Speaking of rasterizing: the 1-hectare data can again be imported by using a specialized function. For example, to import the number of immigrants attribute, we can use this command:

immigrants_100m <- z11_get_100m_attribute("DEM_STAATSANGE_KURZ_2", data_source = con)
#> Fetch attribute from database...
#> Extract coordinates from inspire ID...
#> Transform to raster...

The operation may take a bit longer, as the 1-hectare data are bigger and therefore computationally also a bit more expensive. Fortunately, through their flat data structure, raster data files are usually a bit more comfortable to process. Plotting 1-hectare data is, therefore, also straightforward:

raster::plot(immigrants_100m)

If you wish to work with the data in a vector format, you can also use the as_raster option in the z11::z11_get_100m_attribute() function:

immigrants_100m_sf <- z11_get_100m_attribute("DEM_STAATSANGE_KURZ_2", con, as_raster = FALSE)
#> Fetch attribute from database...
#> Extract coordinates from inspire ID...

Joining Census data to another dataset

Suppose we have a data.frame with 2000 rows, X and Y coordinates and two other variables. We want to link information from the Census to this data.

To get the 100m and 1km INSPIRE IDs from the X and Y coordinates, we can transform the data.frame to an sf object and then use the z11_create_inspire_ids() function:

library(sf)
#> Linking to GEOS 3.6.2, GDAL 2.2.3, PROJ 4.9.3
df <- st_as_sf(df, coords = c("X", "Y"), crs = 3035)
df <- z11_create_inspire_ids(df, column_name = "inspire", combine = TRUE)
head(df) %>% knitr::kable()
